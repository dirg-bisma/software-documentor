# BRD Review Guide

When reviewing a BRD, focus on:
1. **Business Value:** Does solving this problem actually provide value?
2. **Clarity of Objectives:** Can we measure if the objective is met?
3. **Scope Creep Prevention:** Is the 'Out of Scope' section robust enough to prevent future scope creep?
4. **Separation of Concerns:** Reject the document if it dictates a technical solution (e.g., "We need an AWS server").

**Output Format for Review:**
- Executive Summary
- Critical Issues (e.g., missing objectives, technical specs included)
- Missing Information
- Assumptions & Risks
- Recommendations
