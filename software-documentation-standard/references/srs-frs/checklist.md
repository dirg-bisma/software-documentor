# SRS-FRS Checklist

- [ ] Are all functional requirements clearly mapped to PRD features?
- [ ] Are user roles and permissions explicitly defined?
- [ ] Are Non-Functional Requirements (NFRs) measurable?
- [ ] Are core business rules documented?
- [ ] Are requirements testable? (Do they have clear acceptance criteria?)
- [ ] Are there any contradictory requirements?
- [ ] Are assumptions and open questions explicitly stated?
- [ ] Are all requirements uniquely identifiable (e.g., FR-001, NFR-001)?
