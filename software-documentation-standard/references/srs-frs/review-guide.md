# SRS-FRS Review Guide

When reviewing an SRS-FRS, focus on:
1. **Testability:** Can a QA engineer write a test case based purely on the functional requirement and its acceptance criteria?
2. **Completeness:** Are error paths and alternative flows mentioned alongside the happy path?
3. **Traceability:** Is every requirement justified by a product feature?
4. **Ambiguity:** Flag any vague words like "fast", "secure", or "user-friendly".

**Output Format for Review:**
- Executive Summary
- Critical Issues (e.g., untestable requirements, missing NFRs)
- Missing Information
- Contradictions & Ambiguities
- Traceability Gaps
- Recommendations
