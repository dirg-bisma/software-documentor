# FSD Standard

## Purpose
Define *detailed functional behavior*. This document is typically module-specific (e.g., "Reservation Module FSD") and provides the granular details needed by developers to build screens and business logic.

## Must Contain:
- Module behavior overview
- Screen behavior (what happens when a user clicks X)
- Field specifications (data types, mandatory/optional, limits)
- Detailed validation rules
- Workflow behavior and state transitions
- API functional behavior (endpoints expected, functional payloads)
- Integration behavior
- Detailed error behavior and specific error messages
- Batch process or scheduler behavior (if applicable)

## Must NOT Contain:
- Broad product vision
- Database schemas (save for Technical Design)
