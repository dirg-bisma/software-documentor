# FSD Review Guide

When reviewing an FSD, focus on:
1. **Granularity:** Is there enough detail for a front-end or back-end developer to build the feature without guessing?
2. **Edge Cases:** Are validation failures, empty states, and system errors covered?
3. **Consistency:** Does the state machine/workflow match what was defined at a high level in the SRS?
4. **Scope Creep:** Did the FSD introduce new business rules not present in the SRS?

**Output Format for Review:**
- Executive Summary
- Critical Issues (e.g., missing validation rules, undocumented states)
- Missing Information
- Contradictions & Ambiguities
- Traceability Gaps
- Recommendations
