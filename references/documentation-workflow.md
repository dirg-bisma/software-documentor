# Documentation Workflow

Follow this strict step-by-step process before generating any documentation.

1. **Understand Project Context:** Read all available inputs and background information.
2. **Identify Document Type:** Determine if generating a BRD, PRD, SRS, FSD, or Tech Design.
3. **Identify Available Facts:** Extract confirmed information.
4. **Identify Missing Information:** Find gaps that prevent a complete document.
5. **Identify Assumptions:** Explicitly state what is being assumed (use `[ASSUMPTION]`).
6. **Identify Constraints:** Note any business or technical limitations.
7. **Identify Dependencies:** Note external systems, teams, or prior features required.
8. **Determine Scope:** Clearly define IN SCOPE and OUT OF SCOPE.
9. **Generate Document:** Use the appropriate template and standard.
10. **Generate Requirement IDs:** Assign stable IDs (e.g., `BR-001`, `FEAT-001`).
11. **Create Traceability:** Map requirements to their predecessors.
12. **Run Quality Review:** Check completeness, testability, consistency.
13. **Identify Contradictions:** Ensure no conflicting requirements.
14. **Identify Missing Critical Info:** Summarize what must be clarified.
15. **Return Final Output:** Provide Markdown output and the Quality Validation Summary.
