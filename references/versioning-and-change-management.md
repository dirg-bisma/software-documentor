# Versioning and Change Management

Documents are living artifacts. When changes happen, they must be controlled.

## Document Control Block
Every major document must include a Document Control section at the beginning:
- Document ID
- Project Name
- Document Type
- Version (e.g., 1.0, 1.1, 2.0)
- Status
- Author
- Reviewer
- Approver
- Created Date
- Last Updated

## Recommended Statuses
- **Draft:** Work in progress, not ready for review.
- **In Review:** Pending stakeholder approval.
- **Approved:** Baselines established.
- **Rejected:** Did not pass review.
- **Superseded:** Replaced by a newer version.
- **Archived:** No longer relevant.

## Scope Rules
Every major document must clearly define:
- **IN SCOPE:** What is explicitly covered.
- **OUT OF SCOPE:** What is explicitly excluded.

**Detecting Scope Creep:**
If a new requirement is introduced that falls outside the defined IN SCOPE boundaries, the AI must explicitly flag it as a scope change and require a version update or an addendum.
