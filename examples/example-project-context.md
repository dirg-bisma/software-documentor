# Multi-Property Hotel Management Platform Context

**Current Situation:**
- The company operates 8 hotels.
- Each hotel uses a separate, standalone legacy application.
- Data is fragmented. Management cannot easily see consolidated operational and financial information.

**Business Goal:**
- Replace the legacy systems with a centralized platform.
- First production rollout must support at least one hotel, eventually rolling out to all 8.
- The system must support multi-tenant/multi-property data structures from day one.
- Future integrations with payment gateways and third-party OTAs (Online Travel Agencies).

**Challenges:**
- The exact user count, transaction volume, and performance targets are currently unknown and need to be defined.
- Roles (Front Office, Housekeeping, Management) vary slightly between properties but must be standardized.
